#1 HIGHEST Sequential Circuits — Logic Diagram + State Table + State Diagram 99% ▼
Exam 2C Q3 (45 pts) Assig 7 Q1 Final Review Ex. 2 Mid2 Review
★ This was worth 45 points on Exam 2C — the single biggest problem on any exam. It is explicitly listed as #2 and #3 on the professor's "You should know how to" list and appears in every review session. Expect it.
Exact Problem Pattern
Given: two D flip-flops (A, B), 1–2 inputs (X [,Y]), one output (Z/Y) with equations like:
D_A = AX' + XY    ← function of current state + inputs
D_B = AX + BX
Z  = A' + B       ← output equation
Tasks asked: (a) logic diagram, (b) state table, (c) state diagram
Step-by-Step Procedure
  • Logic Diagram: Draw AND/OR/NOT gates feeding D inputs of each flip-flop. Feedback from Q outputs. Clock line to all FFs. Output combinational logic at bottom.
  • State Table: Enumerate all 2² = 4 present states (AB = 00, 01, 10, 11). For each, try each input combination. Compute D_A, D_B by substituting values → these become next state A, B. Compute Z.
  • State Diagram: One bubble per state (labeled "AB"). Arrow for each input, labeled "input/output". Self-loops allowed.

With 1 input X: each state has 2 arrows (X=0 and X=1).
With 2 inputs X,Y: each state has 4 arrows.

Most Likely Variation on Final
Based on Exam 2C pattern, expect 3 sub-parts (a/b/c) each worth ~15 pts. The equations will be similar in complexity to: 
D_A = AX + BX    (Exam 2C)
D_B = A'X + BX
Y  = B
or the review guide's: 
D_A = AX + BX
D_B = A'X + BX
Y  = B
Practice both. The Assig 7 version uses 2 inputs — that may appear too.
⚠ Top Traps — These Cost Points
  • Forgetting the clock line in the logic diagram (lost 2 pts on Exam 2C)
  • Leaving out a transition in the state diagram — every state needs arrows for ALL input combos
  • State diagram label format: input/output, not just input
  • For Mealy machines: output depends on BOTH state and input; for Moore: output depends only on state. Know which one you have
  • When output equation is just Y=B, output column is simply the B flip-flop value — easy points if you remember

... (8 rows total for 2 FFs, 1 input)
#2 HIGHEST Boolean Expr → Logic Diagram (exact) + K-map Simplification + SOP Diagram 99% ▼
Exam 1C Q3 (18 pts) Exam 1C Q4 (20 pts) Final Review Ex. 1 CS42 Review Ex. 1
⚡ This topic appeared as two separate problems on Exam 1C. Expect it again — possibly combined into one multi-part problem worth 25–40 pts on the final.
Exact Problem Pattern (3 parts)
Part (a): Draw logic diagram corresponding EXACTLY to the equation.
         Do NOT simplify. Complements ARE available.
         e.g. F = BC + B(AD + C'D)

Part (b): Simplify with a Karnaugh map. Draw the two-level (SOP)
         logic diagram. Complements ARE available.

Part (c): Redraw using ONLY NAND gates + inverters.
         Complements are NOT available.
Part (a) Shortcut — Exact Diagram
  • Work inside-out: innermost parentheses first
  • Each + (OR) → one OR gate. Each product → one AND gate
  • For F = BC + B(AD + C'D): first build (AD + C'D), then AND with B, then OR with BC
  • Complement inputs (A', B', etc.) come directly as wires — no extra gates needed when they're available
  • Count levels: this tells you the number of gate layers to draw
K-Map Shortcut
  • Convert the expression to a truth table first (fill minterms)
  • Label the K-map correctly: for 3-var (ABC), cols are BC = 00,01,11,10 (Gray code!)
  • For 4-var (ABCD): rows AB = 00,01,11,10 and cols CD = 00,01,11,10
  • Group 1s in powers of 2 (1,2,4,8). Largest groups first. Wrap edges.
  • Each group → one product term. Write simplified SOP.
  • Don't cares (X): can be included in any group to make it larger
Part (c) NAND-Only Shortcut
  • Start from the simplified SOP (sum of products)
  • Replace each AND gate with a NAND gate
  • Replace the final OR gate with a NAND gate (DeMorgan: NAND of complements = OR)
  • Because complements are NOT available: add an inverter on each uncomplemented input going into AND/NAND gates that need the complement
  • Visual check: a 2-level SOP → 2-level NAND-NAND with inverters on raw inputs where needed
⚠ Top Traps
  • "Do not simplify" — students simplify anyway. The diagram must match the equation exactly, even if redundant
  • K-map column/row ordering: must use Gray code (00,01,11,10) — not binary order
  • NAND diagram: when complements are NOT available, you must add an inverter for every complemented literal. Forgetting these = -2 pts each
  • 2-variable K-maps have 4 cells; 3-var → 8 cells; 4-var → 16 cells
#3 HIGH Counter Design with D Flip-Flops 95% ▼
Assig 8 Q1–Q3 Final Review Ex. 4 CS42 Review Ex. 4 (solved) Mid2 Review
★ The professor provided a fully worked solution for the counter sequence 1,3,5,7,9,11,13,15 in the review session notes — complete with state table, K-maps, equations, and logic diagram. This exact problem or a close variation is very likely.
Full Procedure
  • Step 1 — How many flip-flops? 2ⁿ ≥ length of sequence. For 8 states: 4 bits (W,X,Y,Z)
  • Step 2 — State table: List present state, next state. Unused states → don't cares (X)
  • Step 3 — D equations: For each flip-flop, write D_FF = minterm list from next-state column
  • Step 4 — K-maps: One K-map per flip-flop. Include don't cares.
  • Step 5 — Logic diagram: Draw gates + D FFs with clock
★ Memorize This Solved Example (Sequence 1,3,5,7,9,11,13,15)
4 flip-flops: W X Y Z
All are ODD → Z (LSB) is always 1

D_W = WY' + WX' + W'XY    (from K-map)
D_X = XY' + X'Y = X ⊕ Y  (XOR!)
D_Y = Y'                   (just complement Y)
D_Z = 1                    (LSB always 1)

Don't cares: 0,2,4,6,8,10,12,14 (all even numbers)
⚠ Top Traps
  • Forgetting to mark unused states as don't cares → missed simplification = wrong equations
  • State table needs the NEXT state, not the next value in sequence by +1 — it's whatever comes next in the given sequence
  • For 3-bit counters: 3 flip-flops, 8 rows even if sequence is shorter — fill remaining as don't cares
  • K-map size matches number of flip-flops (for W,X,Y,Z → 4-variable K-maps = 16 cells)

#4 HIGH Register Transfer — Logic Diagram with MUX & Load-Enable 90% ▼
Assig 9 Q1–Q3 Final Review Ex. 5 CS42 Review Ex. 5 (fully solved)
Problem Pattern
Given conditional register transfer statements with mutually exclusive control inputs:
C₀: R1 ← R0, R2 ← R0
C₁: R1 ← R0
C₂: R2 ← R1
Only one Cᵢ = 1 at a time.
→ Complete the logic diagram (MUX selects + register LE signals)
Derivation Method

Build a truth table for control outputs (L₀, L₁, L₂ = load enables; S = MUX select):

C₂C₁C₀L₀ (load R0?)L₁ (load R1?)L₂ (load R2?)S (MUX sel)
000000X
0010110
0100100
1000011

From the table: L₀ = 0, L₁ = C₀ + C₁, L₂ = C₀ + C₂, S = C₂

S=0 → MUX routes R0. S=1 → MUX routes R1. The OR gates for L₁ and L₂ are drawn explicitly in the diagram.

★ Assignment 9 Variation — Likely on Final
C₀: R2 ← 0      (clear)
C₁: R2 ← R̄2    (complement)
C₂: R2 ← R1    (transfer)
For this: need AND gates, OR gates, inverters. No MUX needed — the D input expression for each register bit is:
D = C₁·Q̄ + C₂·R1_bit (with C₀ driving clear via the LE or reset)
⚠ Top Traps
  • When no control is active (all C=0), the register must NOT load → LE = 0
  • The MUX output goes to ALL registers' D input — but only the one with LE=1 actually loads
  • For simultaneous transfers (C₀: R1←R0 AND R2←R0), both L₁ and L₂ fire
  • Assig 9 Q2 uses a 4-to-1 MUX (4 sources R0–R3 → R4). Select lines S₁S₀ encode which Cᵢ is active
#5 HIGH MUX Logical Input Mask (4-to-1, find S & B values) 90% ▼
Final Review Ex. 6 CS42 Review Ex. 6 (fully solved)
Problem Pattern
A 4-bit, 4-to-1 MUX circuit where inputs A₀–A₃ and mask B₀–B₃ feed four 4-to-1 MUXes in parallel (one per bit). S₁ is the HIGH-ORDER select bit. Given desired output G₀–G₃, find S₀, S₁, and B₀–B₃.
★ Master Key — Memorize This Table
GoalOperationS₁S₀ selectsB value
G = A (pass through)Select AS = 00 (input 0 = A)irrelevant
G = Ā (complement)XOR with 101 or 11B = 1 (all 1s)
G = 1 (set all bits)OR with 111 (input 3 = 1)B = 1
G = 0 (clear all bits)AND with 010 (input 2 = 0)B = 0

The MUX inputs are wired: input 0 = A (operand), input 1 = A XOR B, input 2 = 0, input 3 = 1. So selecting S₁S₀ = 01 gives A⊕B (logical mask with XOR).

Review session note: "Could B be a mask? AND OR XOR" — think of B as the mask operand.

⚠ Top Traps
  • S₁ is the HIGH-order select bit — the professor bolds this in the problem. S₁S₀ = 10 means input 2, not input 1
  • Each MUX in the 4-bit circuit shares the same S₀, S₁ — but has its own B bit
  • When G = A (identity), B values are irrelevant (don't care) but you still write something
  • When G = Ā, you need S₁S₀ = 01 and B = 1 (XOR with all 1s)
#6 HIGH Bitwise Logic Masks (AND / OR / XOR on n-bit operands) 85% ▼
Assig 9 Q4–Q5 Final Review Ex. 7 CS42 Review Ex. 7 (solved)
★ The Three Rules — Anchor Everything Here
GoalOperationMask BitWhy
Clear bit to 0AND00 AND anything = 0
Set bit to 1OR11 OR anything = 1
Complement/toggle bitXOR11 XOR x = x̄
Leave bit unchangedAND or OR1 (AND) or 0 (OR)Identity element
★ The Exact Problem That Will Appear (Assig 9 Q5 = Review Ex. 7)
Operand: 0011 0101 1100 1010  (16-bit, positions 15–0)

(a) Clear all ODD bit positions to 0
    → AND mask with 0 at odd positions (15,13,11,...,1)
    → Mask: 1010 1010 1010 1010  = 0xAAAA

(b) Set the rightmost 4 bits to 1
    → OR mask with 1 at bits 3,2,1,0
    → Mask: 0000 0000 0000 1111  = 0x000F

(c) Complement the most significant 8 bits
    → XOR mask with 1 at bits 15–8
    → Mask: 1111 1111 0000 0000  = 0xFF00
⚠ Top Traps
  • Bit position 15 = leftmost (MSB), bit position 0 = rightmost (LSB). The professor states this explicitly in the problem — don't flip them
  • "Odd bit positions" means positions 15, 13, 11, 9, 7, 5, 3, 1 (every other bit starting from left for 15, or right for 1). Use the explicitly stated convention
  • Show your work: write out the operand, the mask, and the result bit-by-bit
#7 SOLID Decoder + OR Gate Implementation of Boolean Functions 80% ▼
Final Review Ex. 3 CS42 Review Ex. 3 (solved) Mid2 Review
Problem Pattern
Given: F₁ = AB + A'B'C
        F₂ = AB + A'BC'
        F₃ = AB + AB'C'

Implement with a 3-to-8 decoder and three external OR gates.
Step-by-Step
  • Step 1: Build the truth table for each function (3 inputs A,B,C → 8 rows)
  • Step 2: Identify the minterms where each function = 1
  • Step 3: For the solved example: F₁ = Σm(1,6,7), F₂ = Σm(2,6,7), F₃ = Σm(4,6,7)
  • Step 4: Connect decoder outputs 1,6,7 to OR gate → F₁. Outputs 2,6,7 → F₂. Outputs 4,6,7 → F₃
  • Step 5: Draw: A,B,C → S₂,S₁,S₀ of decoder → 8 outputs → OR gates
⚠ Top Traps
  • Which input is MSB? In a 3-to-8 decoder, S₂ is the most significant (A usually), S₀ least significant (C)
  • A function with output 1 only when AB=11 (regardless of C) covers minterms 6 AND 7 — write both
  • The OR gate has one input per minterm — so a function with 3 minterms needs a 3-input OR gate
#8 LIKELY Adder-Subtractor Circuit (4-bit ripple carry) 75% ▼
Exam 2C Q2 (20 pts) Assig 6 Q5 Mid2 Review
ℹ This appeared on Exam 2C and as a homework problem with the exact same circuit. The professor may include it on the final as a lower-point warm-up or sub-part.
How the Circuit Works
  • S = 0 → ADD. S = 1 → SUBTRACT (2's complement subtraction)
  • Each B bit is XOR'd with S before entering the full adder
  • When S=1: B is complemented, and C₀ = 1, giving A + B̄ + 1 = A - B (2's complement)
  • Output: C₄, S₃, S₂, S₁, S₀
Computation Shortcut
  • S=0: just binary add A + B. C₄ = carry out. Result = S₃S₂S₁S₀
  • S=1: take 2's complement of B, add to A. Show your work in columns (1's complement → add 1)
  • For negative results: the answer is a negative number in 2's complement if C₄=0 when expected otherwise
⚠ Top Traps (from Exam 2C)
  • On Exam 2C, several students lost points for not showing the 2's complement steps explicitly
  • C₄ is the carry OUT of the MSB FA — include it as part of your answer
  • When subtracting and there's no borrow, C₄=1 and the result is positive. When C₄=0, result is negative (2's complement representation)
#9 LIKELY MUX Implementation of Boolean Functions 75% ▼
Exam 2C Q1 (20 pts) Mid2 Review (4-to-1 MUX) Assig 5
Problem Pattern
F(X,Y,Z) = Σm(1,2,4,7)
Implement with one or two 4-to-1 MUXes and a single inverter.
The Trick
  • Use the two most-significant variables (X, Y) as select lines S₁, S₀
  • For each of the 4 groupings of (X,Y): look at F values with Z=0 and Z=1
  • Each data input (I₀–I₃) gets: 0, 1, Z, or Z' based on the function value pattern
  • For 2 functions: use 2 separate 4-to-1 MUXes with shared select lines
Exam 2C Q1 Exact Pattern — Learn This
F = Σm(1,2,4,7), G = Σm(0,1,5,7)

Use X,Y as select → for each group:
XY=00: F: Z=0→0, Z=1→1 → I₀ = Z
XY=01: F: Z=0→1, Z=1→0 → I₁ = Z'  (inverter)
XY=10: F: Z=0→1, Z=1→0 → I₂ = Z'
XY=11: F: Z=0→0, Z=1→1 → I₃ = Z
⚠ Top Traps
  • Must also draw the truth table when asked — don't skip it
  • S₁ is the high-order select bit (connected to X/first variable)
  • The data inputs (0,1,2,3) select when S₁S₀ = 00,01,10,11 respectively
⚠ TRAPS Professor Kurtz's Favorite Tricks & Global Traps Read This ▼
Wording Traps
  • "Exactly" or "Do not simplify" = draw gates matching the equation, every layer, no shortcuts
  • "Complements ARE available" = use A' directly as a wire
  • "Complements are NOT available" = add inverter for each complemented literal
  • "S₁ is the high-order select bit" = always stated when it matters — S₁S₀=10 → input 2, not input 1
Circuit Drawing Traps
  • Sequential circuits: always draw the clock line connecting all D flip-flops
  • Feedback wires: Q outputs must loop back to input gates
  • State diagram: every state needs a transition for EVERY input combination — missing = partial credit deduction
  • Logic diagram: show gate types (AND, OR, NOT — not just boxes)
K-Map Traps
  • Column/row order must be Gray code: 00, 01, 11, 10
  • Groups must be powers of 2: 1, 2, 4, 8, 16
  • Groups can wrap around edges (top↔bottom, left↔right, corners)
  • Don't cares: include in groups to maximize group size — but you don't have to cover them
Arithmetic Traps
  • 2's complement of 0 is 0 (not 2^n)
  • For subtraction: show 2's complement steps explicitly for full credit
  • Overflow vs. carry: C₄=1 doesn't always mean positive result; check sign bits
  • Bit position 15 = MSB (leftmost), bit 0 = LSB (rightmost) — stated explicitly in problems
